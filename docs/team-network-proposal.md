# Подключение команды к общей инфраструктуре

Темы для обсуждения на командной встрече. Цель — за одну встречу договориться: какие сценарии покрываем, какую технологию выбираем, кто администрирует.

Документ готовит Jarvis (AI-агент) для встречи. Если я подключён к встрече — могу пройти по пунктам и помочь зафиксировать решения.

## Контекст

Цеховой PC (RTX 5080, 64GB RAM, Windows 11) становится always-on сервером для общей инфры: Ollama/LLM, scheduled tasks, возможно shared storage. Нужно решить как команда будет получать к нему удалённый доступ, и в целом — как соединять компы команды в единую сеть.

## 1. Сценарии — какие из них нам реально нужны

Прежде чем обсуждать технологии, вычеркнуть лишнее. Для каждого сценария — решить: **нужен / не нужен / позже**.

| # | Кто | Откуда | К чему | Частота |
|---|-----|--------|--------|---------|
| A | Разработчики | Дом / мобильно | Цеховой GPU (Ollama, CUDA) | Ежедневно |
| B | Разработчики | Везде | Shared файлы / репы (если не через git/cloud) | Ежедневно |
| C | Админ | Везде | SSH / RDP на сервер | По ситуации |
| D | Команда | Из офиса | Физическое железо (robot, принтер, калибровочный стенд) | Сессиями |
| E | Внешние (клиенты, аудит) | Из интернета | Подмножество сервисов с авторизацией | Редко |
| F | CI / агенты | Из облака | Memory-server, внутренние API | Постоянно |

## 2. Рамочные решения (не технические — до выбора тулинга)

1. **Модель доверия.** Flat (все видят всех) или ролевая (ACL per scenario)?
2. **Self-hosted vs SaaS.** Свой VPN-сервер или готовое решение (Tailscale / ZeroTier)?
3. **Админ.** Один человек или shared ownership? Что будет если этот человек в отпуске?
4. **Identity.** Личные аккаунты (Google / GitHub SSO) или корпоративный AD / Azure AD?
5. **Бюджет.** $0 / до $X в месяц / неважно?
6. **Compliance.** Есть ли требования к логированию / аудиту / on-prem?

## 3. Варианты решений (матрица)

| Решение | Для сценариев | Плюсы | Минусы | Стоимость |
|---------|---------------|-------|--------|-----------|
| **Tailscale (команда-план)** | A, B, C | Zero-config, ACL, SSO, MagicDNS, работает через любые сети | Vendor lock-in, coordination servers в США | ~$5-6 / user / мес |
| **WireGuard + свой VPS** | A, B, C | Полный контроль, бесплатно | Админ-работа, нет ACL UI, нужно поддерживать | $0 + VPS (~$5/мес) |
| **ZeroTier** | A, B, C | Open-source core, free tier до 25 устройств | Меньше фич чем Tailscale, слабее ACL | $0 на старте |
| **Cloudflare Tunnel / Zero Trust** | E, F | Публичный доступ без open-портов, SSO | Только HTTP(S), не для произвольного TCP | $0 до 50 юзеров, потом ~$7 |
| **Корп VPN (pfSense / OpenVPN)** | A, B, C, D | On-prem контроль, привычно для IT | Тяжёлый setup, ручная поддержка | Стоимость железа |
| **SSH bastion + agent forwarding** | C | Просто, ничего нового учить не надо | Только для техперсонала, не для GUI | $0 |
| **Syncthing / Nextcloud** | B (если не git) | Без VPN, работает в любой сети | Не решает compute / SSH доступ | $0 |

## 4. Что выяснить ДО встречи (инфо-сбор)

Без этого встреча пойдёт по кругу:

- Сколько людей сейчас, сколько будет через 6 месяцев
- OS-сплит (Windows / Mac / Linux / мобильные устройства)
- BYOD или корпоративные устройства
- Уже есть: AD? Google Workspace? Корп-VPN? GitHub Org?
- Цеховой PC физически в чьей сети — личной, арендованной, общей с другими арендаторами?
- Есть ли юридические / compliance требования (например NDA с клиентами)

## 5. Стартовая рекомендация (чтобы встреча шла от чего-то)

**Tailscale (team plan) + Cloudflare Tunnel для публичных сервисов.**

Обоснование:
- Покрывает сценарии A, B, C, E, F минимумом движущихся частей
- SSO из коробки (Google / GitHub / Microsoft)
- ACL позволяет разделить доступ (например: разработчики → только Ollama, админы → всё)
- Работает через любые сети, включая корпоративные с NAT и ограничительными файрволами
- Self-hosted WireGuard — только если compliance требует on-prem или команде критичен $0 бюджет

**Анти-паттерн (обсуждать не надо, сразу отвергнуть):**
- Открывать порты на роутере + basic-auth. Небезопасно, не масштабируется.
- Публичный Ollama без auth. Кто угодно может сжечь GPU.

## 7. Открытые вопросы для дискуссии

- Нужен ли сценарий D (физическое железо) с удалённым управлением, или это всегда on-site работа?
- Доступ к цеховому GPU — для каких задач? Разработка (частое переключение) или production inference (настроил и забыл)?
- Планируется ли расширение железа (ещё компы в цеху, другие локации)? Это влияет на выбор — масштабирование Tailscale против собственной инфры.
