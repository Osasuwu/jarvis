## Решение 1: Где хранить общие настройки?

Нужен единый repo, откуда каждый берёт общий контекст и настраивает под себя.

| Вариант | Плюсы | Минусы | Рекомендация |
|---------|-------|--------|--------------|
| **A) Отдельный repo** `company-org/claude-setup` | Чистое разделение. Каждый клонирует к себе и кастомизирует. Обновления через `git pull`. | Ещё один repo для поддержки. Нужно не забывать пулить. | **Рекомендую для старта** — проще всего начать, переехать потом легко |
| **B) Submodule в каждый проект** | Общие настройки прямо в проекте. Автоматически подтягивается. | Submodule-ад. Конфликты. Новичкам сложно. | Для опытных команд с monorepo |
| **C) Template repo** (GitHub template) | Каждый создаёт свой repo из шаблона, дальше кастомизирует. | После создания — нет sync с оригиналом. Обновления руками. | Если каждый хочет полную независимость |
| **D) Shared directory на ПК** | `~/.claude/` — общие настройки. Sync через скрипт/Git. | Не работает кросс-ОС. Нет version control "из коробки". | Не рекомендую — хрупко |

**Мой совет**: Вариант A — отдельный repo. Каждый клонирует, настраивает. Обновления общих правил — через PR в общий repo.

## Решение 2: Что должно быть ОБЩИМ для всех?

Это ядро — то, что одинаково у каждого. Всё остальное — личное.

### Обязательное (предлагаю включить):

| Компонент | Зачем | Пример |
|-----------|-------|--------|
| **CLAUDE.md** (компания) | Claude знает кто мы, терминологию, ценности | "Мы — [компания]. Основной продукт: [X]. Клиенты: [Y]" |
| **Терминология** | Единые термины для всей команды | "Используем 'клиент', не 'пользователь'. 'Проект', не 'задание'" |
| **Anti-patterns** | Чего Claude НЕ должен делать | "Не предлагай решения без понимания контекста" |
| **Security rules** | Что Claude не должен видеть/делать | "Никогда не включай API ключи в код. Не пуш в main без PR" |

### На обсуждение:

| Компонент | За | Против | Вопрос для команды |
|-----------|-----|--------|--------------------|
| **Общие skills** (/review-pr, /summarize) | Экономят время. Одинаковое качество. | Не всем нужны одни и те же skills | Какие 3-5 skills нужны ВСЕМ? |
| **Общий MCP: GitHub** | Issues, PR из Claude. Все используют GitHub. | Нужен token каждого. Настройка per-user. | Стоит ли включать в base? |
| **Общий MCP: Firecrawl/Web** | Исследование, парсинг документации | Платный API. Не всем нужно. | Включить для разработчиков? |
| **Hooks** (SessionStart) | Контекст при старте сессии | Зависит от рабочего окружения | Какой контекст полезен всем? |
| **Стиль общения Claude** | Единообразие: краткий/детальный, RU/EN | Каждый привык по-своему | Есть ли общий стиль? |

## Решение 3: Какие роли выделить?

Не нужно делать роль для каждого человека. Нужны 2-3 шаблона, которые покрывают всех.

### Предлагаемые роли:

**Developer** — для 4 программистов:
- Skills: /review-pr, /investigate-bug, /gen-migration, /write-test
- MCP: GitHub, Context7 (документация библиотек), Firecrawl (web research)
- Hooks: git status при старте, lint при коммите
- CLAUDE.md: стек проекта, конвенции кода, архитектура

**Business** — для работы с клиентами, документами, идеями:
- Skills: /write-doc, /summarize, /brainstorm, /prepare-meeting
- MCP: GitHub (issues), возможно Notion
- CLAUDE.md: tone of voice, клиентская терминология, шаблоны документов

**Minimal** — для любой роли (бухгалтер и т.д.):
- Только базовый CLAUDE.md компании
- Без MCP серверов (работает через claude.ai/chat + Projects)
- Skills: /summarize, /translate

### Вопрос для команды:
Хватит ли этих 3 ролей? Или кому-то нужно что-то специфическое?

## Решение 4: Как обновлять общие настройки?

| Вариант | Как работает |
|---------|-------------|
| **A) PR в общий repo** | Кто-то предлагает изменение → PR → обсуждение → merge → все пуллят |
| **B) Один ответственный** | Пётр (или кто назначен) обновляет, остальные пуллят |
| **C) Ротация** | Каждую неделю/месяц — другой человек добавляет улучшения |

**Мой совет**: Вариант B для старта (один человек, быстро). Перейти на A когда все освоятся.

## Решение 5: Как подключить людей, не использующих Claude Code?

Claude Code — это терминал. Не-программисты скорее используют claude.ai.

| Инструмент | Для кого | Как использует общий контекст |
|------------|----------|-------------------------------|
| **Claude Code** (CLI/VS Code) | Разработчики | Читает CLAUDE.md автоматически |
| **claude.ai Projects** | Бизнес-роли | Загрузить CLAUDE.md как Project Knowledge |
| **claude.ai + Skills** | Все | Anthropic Skills (загружаются через Settings > Capabilities) |
| **Claude Desktop** | Продвинутые | MCP серверы работают и здесь |

**Ключевой момент**: Один и тот же CLAUDE.md работает и в CLI, и как Project Knowledge в claude.ai. Не нужны разные системы.
