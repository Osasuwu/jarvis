# Phase 3: Базовые инструменты

> Реализация 6 основных инструментов для работы с файлами, оболочкой и веб-ресурсами

**Статус:** ✅ Завершено (16 января 2026)  
**Версия:** v0.3.0

## 📋 Обзор

Phase 3 добавляет в Jarvis набор базовых инструментов, которые позволяют агенту взаимодействовать с файловой системой, выполнять команды и обращаться к интернет-ресурсам.

## 🛠️ Реализованные инструменты

### 1. `file_read` — Чтение файлов

**Описание:** Безопасное чтение текстовых файлов из рабочей директории

**Параметры:**
- `path` (string, обязательный) — относительный путь к файлу
- `encoding` (string, опционально) — кодировка (по умолчанию utf-8)
- `max_bytes` (integer, опционально) — максимальный размер файла (по умолчанию 1,000,000 байт)

**Результат:**
- `success: true` — содержимое файла в `output`
- `success: false` — ошибка с описанием в `error`

**Пример использования:**
```python
from jarvis.tools.builtin import FileReadTool

tool = FileReadTool()
result = await tool.execute(path="src/main.py")
if result.success:
    print(result.output)  # Содержимое файла
```

**Ограничения безопасности:**
- Чтение ограничено рабочей директорией (предотвращение escape-атак)
- Максимальный размер файла 1 МБ по умолчанию
- Risk level: **LOW**

---

### 2. `file_write` — Запись файлов

**Описание:** Безопасная запись или дополнение текстовых файлов

**Параметры:**
- `path` (string, обязательный) — относительный путь к файлу
- `content` (string, обязательный) — содержимое для записи
- `append` (boolean, опционально) — дополнить вместо перезаписи (по умолчанию false)
- `encoding` (string, опционально) — кодировка (по умолчанию utf-8)

**Результат:**
- `success: true` — файл записан, путь в `output`
- `success: false` — ошибка с описанием в `error`

**Пример использования:**
```python
from jarvis.tools.builtin import FileWriteTool

tool = FileWriteTool()
result = await tool.execute(
    path="output.txt",
    content="Hello, World!",
    append=False
)
```

**Ограничения безопасности:**
- Автоматически создает родительские директории
- Ограничено рабочей директорией
- **Требует подтверждения** перед выполнением (risk level: **MEDIUM**)

---

### 3. `list_directory` — Список файлов

**Описание:** Безопасный список содержимого директории

**Параметры:**
- `path` (string, опционально) — директория (по умолчанию текущая)
- `show_hidden` (boolean, опционально) — показывать скрытые файлы (по умолчанию false)
- `max_items` (integer, опционально) — максимум элементов (по умолчанию 200)

**Результат:**
- `success: true` — список элементов в `output` (массив объектов)
- `success: false` — ошибка

**Структура элемента:**
```json
{
  "name": "file.txt",
  "type": "file",  // или "dir"
  "size": 1024     // размер в байтах (0 для директорий)
}
```

**Пример использания:**
```python
result = await tool.execute(path="src", max_items=50)
if result.success:
    for item in result.output:
        print(f"{item['type']:4} {item['name']:30} {item['size']:10}")
```

**Ограничения безопасности:**
- Ограничено рабочей директорией
- Максимум 200 элементов по умолчанию
- Risk level: **LOW**

---

### 4. `shell_execute` — Выполнение команд

**Описание:** Выполнение системных команд в оболочке

**Параметры:**
- `command` (string, обязательный) — команда для выполнения
- `timeout` (integer, опционально) — таймаут в секундах (по умолчанию 15)

**Результат:**
- `success: true` — команда выполнена, вывод в `output`
- `success: false` — ошибка выполнения в `error`

**Пример использования:**
```python
result = await tool.execute(command="echo 'Hello' && date")
if result.success:
    print(result.output)  # Вывод команды
```

**Ограничения безопасности:**
- Выполняется в рабочей директории
- Таймаут предотвращает зависания (15 сек по умолчанию)
- Выводит stdout и stderr
- **Требует подтверждения** (risk level: **HIGH**)

---

### 5. `web_fetch` — Загрузка URL

**Описание:** Получение содержимого веб-страницы по URL

**Параметры:**
- `url` (string, обязательный) — URL для загрузки
- `timeout` (integer, опционально) — таймаут в секундах (по умолчанию 10)

**Результат:**
- `success: true` — содержимое в `output` (для текста/JSON) или размер в байтах
- `success: false` — ошибка сети/таймаута

**Пример использования:**
```python
result = await tool.execute(url="https://example.com/api")
if result.success:
    print(result.output)  # HTML, JSON или размер
```

**Ограничения:**
- Таймаут 10 секунд
- Автоматически декодирует текст/JSON
- Для бинарных данных показывает размер
- Risk level: **LOW**

---

### 6. `web_search` — Поиск в интернете

**Описание:** Поиск в интернете через DuckDuckGo Instant Answer API (без API ключа)

**Параметры:**
- `query` (string, обязательный) — поисковый запрос
- `max_results` (integer, опционально) — максимум результатов (по умолчанию 5)

**Результат:**
- `success: true` — результаты в `output` (массив объектов)
- `success: false` — ошибка или нет результатов

**Структура результата:**
```json
[
  {
    "title": "Описание результата",
    "url": "https://example.com"
  }
]
```

**Пример использования:**
```python
result = await tool.execute(query="python asyncio tutorial", max_results=5)
if result.success:
    for item in result.output:
        print(f"{item['title']}\n  {item['url']}\n")
```

**Преимущества:**
- Бесплатный поиск (не требует API ключа)
- Быстрый Instant Answer API
- Risk level: **LOW**

## 🔒 Система Risk Levels

Инструменты классифицированы по уровню опасности:

| Tool | Risk Level | Требует подтверждение | Описание |
|------|------------|----------------------|----------|
| `file_read` | LOW | ❌ | Только чтение, безопасно |
| `file_write` | MEDIUM | ✅ | Модификация файловой системы |
| `list_directory` | LOW | ❌ | Только чтение |
| `shell_execute` | HIGH | ✅ | Выполнение произвольных команд |
| `web_fetch` | LOW | ❌ | Только получение данных |
| `web_search` | LOW | ❌ | Только поиск |

Инструменты с HIGH/MEDIUM риском будут требовать подтверждения пользователя перед выполнением (реализуется в Phase 4: Human-in-the-Loop).
