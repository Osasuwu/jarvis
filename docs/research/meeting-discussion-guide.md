# Гайд для обсуждения: апгрейд AI-процессов

> Дата: 8 апреля 2026
> Контекст: все уже активно используют Claude. 3 разработчика + техническая команда (3D, инструменты, контент).
> Формат: ~30-40 мин после презентации
> Подготовка: все заполнили опрос, Пётр видит ответы

---

## Стратегия ведения

**Все уже в деле.** Команда маленькая, все early adopters. Цель обсуждения:
- Обменяться находками (каждый нашёл что-то, чего другие не пробовали)
- Найти дублирование (где каждый изобретает велосипед)
- Договориться, что делаем общим
- Понять что нужно техн. команде (3D, инструменты) — у них другие задачи

---

## Блок 1: Обмен находками (10 мин)

> Каждый по 1-2 минуте. Записывать — здесь всплывают неочевидные паттерны.

### "Какой самый полезный приём работы с Claude вы нашли?"

### "Что объясняете Claude каждую сессию заново?"

Цель: кандидаты для CLAUDE.md. Если 2 из 3 объясняют одно и то же — это общий контекст.

### "Когда Claude навредил? Потратили время на исправление?"

Цель: кандидаты для anti-patterns в CLAUDE.md.

---

## Блок 2: Где теряется эффективность (10 мин)

### "Ваши Claude-сессии знают о работе друг друга?"

Скорее всего нет. Каждый в изолированном контексте. Это аргумент за shared CLAUDE.md.

### "Было такое что один уже решил проблему с Claude, а другой потратил время на то же самое?"

Конкретные примеры дублирования — это ROI для общей системы.

### "Что бы изменилось, если бы Claude знал ВСЕ решения команды?"

Не "AI memory system", а "Claude помнит что коллега решил на прошлой неделе".

### Для техн. команды: "Какие AI-инструменты нашли за последний месяц? Как делитесь находками?"

Цель: эти люди активно ищут новые инструменты. Если находки остаются в голове одного человека — потеря value для всех.

---

## Блок 3: Что делаем общим (10 мин)

### "Какие задачи повторяются у каждого?"

Для разработчиков: review PR, написание тестов, расследование бага, описание PR.
Для техн. команды: поиск инструментов, сравнение решений, подготовка презентаций.
Это кандидаты для shared skills.

### "Какие внешние инструменты переключаете во время работы?"

GitHub, трекер, Slack, Blender, Figma — что реально используют? Кандидаты для MCP интеграций.

### "Как выглядит идеальное начало рабочего дня? Что должно быть готово?"

Кандидат для SessionStart hook / утреннего дайджеста.

---

## Блок 4: Конкретные решения (10 мин)

### "CLAUDE.md — что обязательно должно быть?"

Один человек пишет первую версию → остальные ревьюят. Обсудить минимум: стек, конвенции, anti-patterns, терминология.

### "Какие 3-5 общих команд нужны всем?"

Пройтись: /review-pr, /investigate-bug, /write-test, /summarize, /write-doc. Что ещё?

### "Как обновляем общие настройки?"

Один ответственный или PR от любого? Решить сейчас, иначе протухнет.

### "Что техн. команда хочет от общей системы?"

У них другие задачи — 3D, контент, инструменты. Что из общей инфраструктуры им полезно, а что нет?

## Чего не делать

- Не показывать личный AI-агент целиком (multi-agent, PM layer) — это будущее, сейчас рано
- Показать 1-2 демо: CLAUDE.md + один skill
- Не обещать конкретные сроки — "попробуем неделю, посмотрим"
- Не разделять обсуждение на "для девов" и "для остальных" — все вместе, но задавать вопросы каждому сегменту
