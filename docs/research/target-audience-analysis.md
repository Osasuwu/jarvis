## 1. Состав команды

| Роль | Кол-во | AI usage | Основные задачи |
|------|--------|----------|----------------|
| Разработчики | 3 (включая Петра) | Claude Code, claude.ai | Код, архитектура, review, дебаг |
| Техническая команда (не код) | ? (уточнить) | claude.ai, AI для 3D/контента | 3D, инструменты, фреймворки, презентации, адаптация AI |

**Все — early adopters.** Деления на скептиков/энтузиастов нет. Вся команда активно использует AI.

---

## 2. Два сегмента (по типу работы, не по готовности)

### Сегмент A: Разработчики (3 человека)
- **Инструменты:** Claude Code / VS Code extension, GitHub
- **Потребности:** shared context проектов, автоматизация code review/tests/docs, delegation задач
- **Quick wins:** CLAUDE.md, shared skills (/review-pr, /investigate, /write-test), MCP GitHub
- **Уровень настройки:** от "использую из коробки" до "продвинутая кастомизация" (уточнить через опрос)

### Сегмент B: Техническая команда (не код)
- **Инструменты:** claude.ai, AI для 3D (Meshy, Tripo, Luma?), AI для контента/презентаций
- **Потребности:** единый контекст компании/проектов, AI для поиска и оценки инструментов, генерация контента
- **Quick wins:** claude.ai Projects с CLAUDE.md компании, role-based промпты
- **Особенность:** активно ищут и адаптируют новые AI инструменты — могут быть источником находок для всей команды

## 3. Jobs to be Done

### Общие (оба сегмента)
| Job | Текущее решение | Желаемое |
|-----|----------------|----------|
| Не объяснять контекст заново | Повторять в каждой сессии | Shared CLAUDE.md, AI помнит |
| Не терять решения | В голове / нигде | Shared memory / documented decisions |
| Не дублировать работу | Каждый ищет сам | Общая база находок, shared prompts |
| Быстро оценить новый инструмент | Вручную, каждый по-своему | AI + structured evaluation |

### Разработчики
| Job | Текущее | Желаемое |
|-----|---------|----------|
| Не ждать code review | Ждать коллегу | AI pre-review + human final |
| Не писать boilerplate | Копипаст | AI генерирует по контексту проекта |
| Автоматизировать PM-рутину | Вручную | AI triage, delegation, monitoring |

### Техническая команда
| Job | Текущее | Желаемое |
|-----|---------|----------|
| Найти лучший инструмент для задачи | Google + пробовать каждый | AI structured research + comparison |
| Сделать презентацию | Вручную | AI draft + human polish |
| Адаптировать новый AI tool | Trial and error | Shared playbook, documented findings |

## 5. Стратегия rollout

Нет фаз "pilot → expand". Команда маленькая, все в деле.

| Неделя | Что | Для кого |
|--------|-----|----------|
| 0 | Опрос + собрание | Все |
| 1 | CLAUDE.md + skills + setup | Разработчики |
| 1 | claude.ai Projects + role prompt | Техническая команда |
| 2 | GitHub Actions (auto-review) | Разработчики |
| 2 | Shared findings playbook | Все |
| 3-4 | Измерение + корректировка | Все |
| 5+ | AI PM layer, delegation | Разработчики (+ возможно координация для техн. команды) |
