# Исследование: Claude — не джун-исполнитель, а инфраструктура

**Цель:** Побудить команду (5 разработчиков) переосмыслить, как они используют Claude Max
**Дата презентации:** 8 апреля 2026
**Ключевой тезис:** Claude можно настроить так, что он перестаёт быть "машиной, выполняющей команды" и становится частью вашей рабочей инфраструктуры

---

## Часть 1: Три уровня использования AI

### Уровень 1: Чат-исполнитель 🟡
> "Напиши функцию, которая делает X"

- Claude как джун: получает задачу → пишет код → ты проверяешь
- Каждая сессия — с нуля, без контекста проекта
- Ты тратишь время на объяснение контекста каждый раз
- **Большинство людей застревают здесь**

### Уровень 2: Настроенный инструмент 🟢
> "Claude знает наш проект, наши стандарты, наши инструменты"

- CLAUDE.md — Claude сразу знает архитектуру, стандарты, anti-patterns
- Кастомные команды — `/review`, `/test`, `/doc` вместо повторных промптов
- MCP серверы — Claude работает с GitHub, базой, Slack напрямую
- Хуки — автоформат, линтинг, защита файлов автоматически
- **Один раз настроил — работает для всей команды**

### Уровень 3: Персональная инфраструктура 🔵
> "Claude — не инструмент, а коллега с памятью, инициативой и специализацией"

- Субагенты — специализированные AI под конкретные задачи
- Память между сессиями — Claude помнит решения, контекст, предпочтения
- Scheduled tasks — Claude работает по расписанию без вашего участия
- Собственные процессы — triage, research, risk-radar, self-review
- **Пример: наш реальный кейс**

## Часть 2: Реальные кейсы — что люди построили

### Кейс 1: наш собственный

Персональный AI-агент, построенный поверх Claude Code:

| Возможность | Что делает |
|------------|-----------|
| **Идентичность (SOUL.md)** | Не ассистент, а старший инженер с мнениями. Спорит, предлагает лучшие решения |
| **Triage** | Автоматически проверяет GitHub issues: стейлые, заблокированные, без метаданных |
| **Risk Radar** | Сканирует CI, security alerts, overdue milestones |
| **Research** | Исследует тему, сохраняет результат в память. Может работать в фоне |
| **Nightly Research** | Каждую ночь ищет новости по AI/MCP/Claude, релевантные проекту |
| **Intel** | Дайджест новых разработок в области AI-агентов |
| **Self-improve** | Анализирует собственный код, находит улучшения, реализует их |
| **Delegate** | Получает issue → реализует end-to-end в отдельном worktree |
| **Reflect** | Ревью прошлых решений: что сработало, что нет, уроки |
| **Память (Supabase)** | Запоминает решения, ошибки, предпочтения МЕЖДУ сессиями на ВСЕХ устройствах |

**Ключевое:** это не фреймворк, не сложная разработка. Это набор markdown-файлов + MCP сервер для памяти. Каждый skill — это файл `SKILL.md`.

### Кейс 2: RonOS (Ron Forbes, PM в Meta Quest)

Не программист построил **персональную операционную систему**:
- Утренние/вечерние рутины с приоритизацией
- Обработка заметок с митингов → саммари + action items
- За 30 дней: **2000+ организованных заметок**
- Инструменты: Claude Code + Obsidian. Настройка: **15 минут**

### Кейс 3: Wyndo (контент-автор)

Claude Code как **ОС для исследований и контента**:
- Slash-команды: `/quick-edit`, `/generate-ideas`
- Субагенты мониторят AI-новости в фоне
- MCP: Perplexity (реал-тайм поиск) + Firecrawl (скрейпинг)
- Мобильный доступ: через GitHub Issues с телефона → Claude обрабатывает

### Кейс 4: ChatOps-платформа (DevOps инженер)

**9 MCP серверов + 150 инструментов** = Claude управляет инфраструктурой:
- NetBox: поиск устройств, VM, IP, VLAN
- Proxmox: управление виртуалками (кастомный MCP)
- Всё через чат: "покажи все VM на node-2" → Claude идёт в Proxmox API

### Кейс 5: RFC-ориентированная разработка

Claude Code для **процессов**, не кода:
- `/rfc [topic]` → дизайн-документ + цикл ревью + фиксация решения
- `/postmortem [incident]` → таймлайн, root cause, action items
- Tech Debt Registry — отслеживание отложенных решений
- `/archive` — автоочистка старых записей для экономии токенов

## Часть 3: Что можно подключить снаружи (MCP + интеграции)

### MCP серверы — Claude получает "руки"

MCP = стандартный протокол, через который Claude может взаимодействовать с любым внешним сервисом.

| Категория | MCP-сервер | Что даёт разработчику |
|-----------|-----------|----------------------|
| **Код** | GitHub | Issues, PRs, code search — без переключения в браузер |
| **Данные** | PostgreSQL, Supabase | Read-only запросы для дебага прямо из Claude |
| **Коммуникация** | Slack | Поиск обсуждений для контекста задачи |
| **Проекты** | Linear, Jira, Notion | Контекст из тикетов и документации |
| **Мониторинг** | Sentry, Datadog | Логи ошибок и метрики прямо в контексте |
| **Документация** | Context7 | Актуальные доки библиотек (вместо устаревших знаний модели) |
| **Инфра** | Docker, Proxmox, NetBox | Управление контейнерами и инфраструктурой |
| **Файлы** | Filesystem, Google Drive | Доступ к файлам за пределами рабочей директории |
| **Браузер** | Playwright, Puppeteer | Автоматизация браузера, тестирование UI |
| **Кастомные** | Ваш внутренний API | Любой REST/GraphQL API можно обернуть в MCP |

**Настройка:** один файл `.mcp.json` в корне проекта, шарится через git.

### Что ещё кроме MCP

| Интеграция | Как работает |
|-----------|-------------|
| **GitHub Actions + Claude** | Claude как ревьюер в CI: автоматический pre-review каждого PR |
| **Scheduled tasks** | Claude по крону: ночной research, утренний дайджест, еженедельный аудит |
| **Claude Desktop + Chrome** | Управление браузером для тестирования, скрейпинга, заполнения форм |
| **Obsidian / VS Code** | Claude работает с вашими заметками и файлами как с кодовой базой |
| **Telegram/Slack бот** | Claude отвечает на вопросы команды в чате (через API) |

## Часть 4: Не только код — что ещё делает разработчик

Программирование — это ~40% работы разработчика. Остальное:

| Задача | Как Claude помогает | Экономия |
|--------|-------------------|----------|
| **Code review** | Pre-screening PR на стиль, безопасность, баги | 30-50% |
| **Написание тестов** | Генерация из существующего кода | 40-60% |
| **Документация** | API docs, README, архитектурные описания | 50-70% |
| **Дебаг** | Анализ стектрейсов, поиск паттернов | 30-50% |
| **RFC / ADR** | Дизайн-документы, фиксация архитектурных решений | 40-60% |
| **Онбординг** | Новичок спрашивает Claude вместо отвлечения сениоров | 40-60% |
| **Коммит-сообщения** | Генерация из diff | 70-90% |
| **Исследование** | Сравнение технологий, чтение документации | 30-50% |
| **Инцидент-менеджмент** | Postmortem, таймлайны, action items | 40-50% |
| **Ежедневная рутина** | Утренний дайджест, приоритизация, заметки с митингов | 30-40% |

## Часть 5: Честные цифры

### Что говорят исследования (не маркетинг)

| Исследование | Результат |
|-------------|-----------|
| METR (РКИ, 2025) | Разработчики думали что на 20% быстрее, на самом деле на **19% медленнее** |
| Thoughtworks | ~8% нетто по всему циклу |
| McKinsey (зрелые команды) | 16-30% прирост |
| Faros Engineering | +21% задач, но +91% время code review |
| SonarSource | AI-код содержит на 9% больше багов |

### Почему "3x" всё-таки возможно

Не через ускорение кода, а через **устранение целых категорий работы**:

```
Было: 40% код + 20% review + 15% docs + 10% debug + 15% рутина = 100%
Стало: 40% код + 5% review-надзор + 3% docs-проверка + 5% debug + 2% рутина = 55%
Освободившиеся 45% → на новые фичи, архитектуру, обучение
```

### Что не измерять
- ❌ Строки кода (AI раздувает)
- ❌ Количество PR (больше PR ≠ больше ценности)
- ❌ Скорость набора (без качества — бессмысленно)

### Что измерять
- ✅ Cycle time (commit → deploy)
- ✅ Escaped defects (баги в проде)
- ✅ Code churn (сколько AI-кода переписывается)
- ✅ Субъективная удовлетворённость команды

## Часть 6: Антипаттерны — ловушки, в которые попадают все

| Ловушка | Почему опасна | Что делать |
|---------|-------------|-----------|
| Доверие без ревью | +9% багов, +322% уязвимостей | AI-код = PR джуниора. Всегда проверяй |
| Каждая сессия с нуля | Тратишь 5 мин на контекст × 10 раз/день = час | CLAUDE.md + память |
| AI для всего | 61% разработчиков никогда не юзают agentic mode | Знай топ-10 задач, применяй только там |
| "Vibe coding" без плана | Непоследовательный код | Spec → план → код |
| Секреты в промптах | Утечка | Политика + secret scanning |

## Часть 7: С чего начать (quick wins)

### Для каждого (5 минут)
1. Добавь `CLAUDE.md` в свой проект — опиши архитектуру и стандарты
2. Используй `/commit` для коммит-сообщений

### Для энтузиастов (30 минут)
3. Настрой хук автоформатирования (PostToolUse)
4. Подключи GitHub MCP — issues и PR без браузера
5. Создай 2-3 кастомные команды для повторяющихся задач

### Для продвинутых (2-4 часа)
6. Настрой субагентов для code review и тестирования
7. Подключи MCP к вашей базе данных (read-only)
8. Создай персональный workflow: утренний дайджест, triage, research

## Часть 8: Вопросы для обсуждения

> Цель: каждый уходит с 1-2 идеями, что автоматизировать у себя

1. **Какую рутину вы делаете каждый день, которую стыдно автоматизировать?**
   (коммит-сообщения, описания PR, обновление тикетов)

2. **Какой контекст вы повторно объясняете Claude каждую сессию?**
   (→ это кандидат для CLAUDE.md)

3. **Какие внешние инструменты вы используете параллельно с кодом?**
   (Slack, Jira, база данных → кандидаты для MCP)

4. **Что бы вы автоматизировали, если бы Claude помнил контекст между сессиями?**

5. **Какие процессы в команде можно описать как skill?**
   (деплой-чеклист, onboarding-гайд, incident response)
